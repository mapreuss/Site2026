## 🧠 Assistente de triagem de tickets com GPT
    
### 🎯 O Desafio
As equipes de produto e suporte estavam sobrecarregadas com o volume de solicitações recebidas de forma desestruturada. O processo de triagem era manual, repetitivo, e sujeito a erros — resultando em tickets duplicados, atrasos no encaminhamento e retrabalho para engenharia.

### 🔍 Restrições da Equipe
- Alto volume de entradas vagas ou sem contexto.  
- Dificuldade em diferenciar bugs de feature requests desde o início.  
- Trabalho manual para verificar se o problema já estava registrado.  
- Requisito de integração com ferramentas já em uso: GitHub, Jira, Zendesk.

### 💡 Minha Solução
Desenvolvi do zero um **assistente GPT personalizado** para triagem automatizada de tickets. Utilizei outros GPTs como copilotos para criar o prompt principal e o fluxo de decisão.

A solução:  
✅ Entende descrições em linguagem natural  
✅ Classifica o problema como bug ou feature request  
✅ Busca por tickets similares no GitHub e Jira  
✅ Inicia um fluxo guiado de criação de ticket, apenas se necessário

Construí uma prova de conceito conectando aos repositórios reais do time de Product Ops. Essa POC servirá como base para expansão futura com suporte e produto.

### 📊 O Impacto
- 📌 Projeto em fase de testes, com POC funcional  
- 🤝 Equipes demonstraram entusiasmo, desde que integradas às ferramentas atuais  
- 🧪 Ponto de partida validado para priorização colaborativa de funcionalidades  
- 🔄 Alinhamento inicial com suporte e produto para evolução do escopo'
